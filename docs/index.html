Scoring Methodology
How our engine ranks instances. The EC2 Instance Advisor uses Multi-Criteria Decision Analysis (MCDA) — an established operations-research framework — to produce objective, bias-free recommendations.
🎯
Weighted Scoring Model (WSM)
Each EC2 instance is evaluated across five criteria: price, vCPUs, memory, network, and GPU count. Your slider inputs directly set the mathematical weight for each criterion.
Composite Score =
(wprice × NormPrice) + (wcpu × NormCPU) +
(wmem × NormMem) + (wnet × NormNet) +
(wgpu × NormGPU)

Where each wi = slider_value / sum_of_all_sliders
📊
Metric Normalization
Raw metrics span vastly different scales (e.g., $0.01/hr vs 768 GiB RAM). Each is min-max normalized to [0, 1] within the active filter set. Hardware metrics use direct normalization (higher = better); price uses inverse (lower = better).
Direct: score = (value − min) / (max − min)
Inverse: score = 1 − (value − min) / (max − min)

A $0.01/hr instance scores high on price; a 96-vCPU instance scores high on compute.
⚖️
User-Defined Weights
Sliders (0–100) are normalized to weights summing to 1.0. Only the relative priority between criteria matters, not absolute values. Price=70, CPU=10 produces the same ranking as Price=7, CPU=1.
Example — Balanced Preset:
Price: 35 → 0.35  ·  CPU: 25 → 0.25  ·  Mem: 20 → 0.20
Network: 10 → 0.10  ·  GPU: 10 → 0.10

Sum = 100 → Each weight = slider / 100

Workload → Family Quick Reference
Match your primary bottleneck to the right EC2 family. Use this alongside the recommendation above.
Workload Type Recommended Family Characteristics
Web & App Servers General Purpose (M/T) Balanced CPU and memory. T-series for burstable dev/test; M-series for steady production traffic.
High-Traffic APIs & HPC Compute Optimized (C) High vCPU-to-memory ratio. Scientific modeling, batch processing, and dedicated game servers.
In-Memory Databases Memory Optimized (R/X) 8:1+ RAM-to-vCPU ratio. Redis, SAP HANA, real-time analytics, large JVM heaps.
NoSQL & Data Warehousing Storage Optimized (I/D) Local NVMe SSDs for millions of low-latency IOPS. MongoDB, Cassandra, high-speed logging.
AI Training & Inference Accelerated (P/G/Trn/Inf) P-series for GPU training; G-series for inference & rendering; trn1/inf2 for AWS ML silicon.
