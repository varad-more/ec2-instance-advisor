How to choose the right instance
Step 1 — Workload type
CPU for builds & APIs, GPU for ML/AI, Memory for large caches & JVM, Storage for high-IOPS NVMe, General for balanced apps.
Step 2 — Pick region
Keep compute close to users and data. Use the Regional Price chart to compare costs across AWS regions.
Step 3 — Use presets
Jump-start with Balanced, Cost First, Performance, or ML/AI — then fine-tune sliders for your exact priorities.
Step 4 — Read the charts
Line chart = price-performance curves per category. Radar = dimension strengths of top 5. Bar = metric comparison. Regional = cheapest region for #1. Click any table row to see a detailed comparison.
Step 5 — Check the score breakdown
Bars in the recommendation card show each dimension's raw normalized score — see exactly why an instance ranked #1.
Step 6 — Validate in production
Benchmark with your real workload and monitor utilization before committing to an instance family.
